물리 계층¶

개요¶

물리 계층(Physical Layer)은 OSI 모델의 최하위 계층으로, 비트(0과 1)를 실제 물리적 신호(전기, 광, 무선)로 변환하여 전송하는 역할을 담당합니다. 이 레슨에서는 물리 계층의 역할, 전송 매체의 종류, 신호의 특성, 대역폭과 전송 속도, 그리고 실제 케이블과 커넥터의 종류를 학습합니다.

난이도: ⭐ (기초)

물리 계층의 주요 기능¶

┌─────────────────────────────────────────────────────────────────┐
│                    물리 계층의 주요 기능                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   1. 비트 동기화 (Bit Synchronization)                          │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │   송신자와 수신자의 클럭을 동기화하여 비트를 정확히     │   │
│   │   인식할 수 있게 함                                      │   │
│   │                                                          │   │
│   │   송신 클럭: ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐                       │   │
│   │             └─┘ └─┘ └─┘ └─┘ └─┘                       │   │
│   │   수신 클럭: ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐  (동기화 필요)       │   │
│   │             └─┘ └─┘ └─┘ └─┘ └─┘                       │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│   2. 비트 전송률 제어 (Bit Rate Control)                        │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │   초당 전송되는 비트 수 (bps) 결정                       │   │
│   │   예: 100 Mbps, 1 Gbps, 10 Gbps                          │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│   3. 물리적 토폴로지 정의                                       │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │   장치 간 물리적 연결 방식 (버스, 스타, 링 등)           │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│   4. 전송 모드 정의                                             │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                                                          │   │
│   │   단방향 (Simplex):                                     │   │
│   │   A ───────────────────────────────► B                  │   │
│   │   (키보드 → 컴퓨터, TV 방송)                            │   │
│   │                                                          │   │
│   │   반이중 (Half-Duplex):                                 │   │
│   │   A ◄──────────────────────────────► B                  │   │
│   │   (한 번에 한 방향만, 무전기)                           │   │
│   │                                                          │   │
│   │   전이중 (Full-Duplex):                                 │   │
│   │   A ◄═══════════════════════════════► B                  │   │
│   │   (양방향 동시, 전화, 이더넷)                           │   │
│   │                                                          │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│   5. 인터페이스 및 매체 정의                                    │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │   - 커넥터 종류 (RJ-45, SFP 등)                         │   │
│   │   - 케이블 유형 (UTP, 광섬유 등)                        │   │
│   │   - 전압 레벨, 신호 타이밍                              │   │

물리 계층의 관련 장비¶

┌─────────────────────────────────────────────────────────────────┐
│                    물리 계층 관련 장비                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   ┌──────────────┬───────────────────────────────────────────┐  │
│   │    장비      │                 기능                       │  │
│   ├──────────────┼───────────────────────────────────────────┤  │
│   │    허브      │ 다중 포트 리피터, 신호 증폭 및 브로드캐스트│  │
│   │   (Hub)      │ 모든 포트로 신호 전달                     │  │
│   ├──────────────┼───────────────────────────────────────────┤  │
│   │   리피터     │ 신호 증폭, 전송 거리 연장                 │  │
│   │  (Repeater)  │ 감쇠된 신호를 원래 형태로 복원            │  │
│   ├──────────────┼───────────────────────────────────────────┤  │
│   │    NIC       │ 네트워크 인터페이스 카드                  │  │
│   │(Network Card)│ 컴퓨터와 네트워크 연결                    │  │
│   ├──────────────┼───────────────────────────────────────────┤  │
│   │    모뎀      │ 디지털 ↔ 아날로그 신호 변환               │  │
│   │   (Modem)    │ 전화선을 통한 데이터 전송                 │  │
│   ├──────────────┼───────────────────────────────────────────┤  │
│   │  미디어      │ 서로 다른 매체 타입 연결                  │  │
│   │  컨버터      │ 예: 광섬유 ↔ UTP 변환                     │  │
│   └──────────────┴───────────────────────────────────────────┘  │
│                                                                  │
│   허브의 동작:                                                  │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                                                          │   │
│   │   PC1 ────┐                                             │   │
│   │           │                                             │   │
│   │   PC2 ────┼──── HUB ───► 모든 포트로 신호 복제         │   │
│   │           │                                             │   │
│   │   PC3 ────┘                                             │   │
│   │                                                          │   │
│   │   * 허브는 L1 장비로 MAC 주소를 이해하지 못함           │   │
│   │   * 받은 신호를 모든 포트로 전달 (브로드캐스트)         │   │
│   │   * 충돌 발생 가능 (하나의 충돌 도메인)                 │   │
│   │                                                          │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                  │

│                                                                  │

│   │                                                          │   │
│   │   데이터가 출발지에서 목적지까지 걸리는 시간             │   │
│   │                                                          │   │
│   │   총 지연 = 전파 지연 + 전송 지연 + 큐잉 지연 + 처리 지연│   │
│   │                                                          │   │
│   │   1. 전파 지연 (Propagation Delay)                      │   │
│   │      - 신호가 매체를 통과하는 시간                      │   │
│   │      - 거리 / 전파 속도                                 │   │
│   │      - 빛의 속도 ≈ 3×10^8 m/s (진공)                   │   │
│   │                                                          │   │
│   │   2. 전송 지연 (Transmission Delay)                     │   │
│   │      - 데이터를 링크에 올리는 시간                      │   │
│   │      - 데이터 크기 / 대역폭                             │   │
│   │                                                          │   │
│   │   3. 큐잉 지연 (Queuing Delay)                          │   │
│   │      - 라우터에서 대기하는 시간                         │   │
│   │      - 네트워크 혼잡도에 따라 변동                      │   │
│   │                                                          │   │
│   │   4. 처리 지연 (Processing Delay)                       │   │
│   │      - 헤더 검사, 라우팅 결정 시간                      │   │
│   │                                                          │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│   RTT (Round Trip Time):                                        │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                                                          │   │
│   │   패킷이 왕복하는 데 걸리는 시간                        │   │
│   │                                                          │   │
│   │   클라이언트 ────────────────────► 서버                 │   │
│   │              ◄────────────────────                      │   │
│   │              ◄──────────────────────►                   │   │
│   │                      RTT                                │   │
│   │                                                          │   │
│   │   측정: ping 명령어                                     │   │
│   │   $ ping google.com                                     │   │
│   │   time=15.2 ms  ← 이것이 RTT                           │   │
│   │                                                          │   │

샤논 용량 공식¶

┌─────────────────────────────────────────────────────────────────┐
│                      채널 용량 공식                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   나이퀴스트 공식 (무잡음 채널):                                │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                                                          │   │
│   │   C = 2 × B × log₂(L)                                    │   │
│   │                                                          │   │
│   │   C: 채널 용량 (bps)                                    │   │
│   │   B: 대역폭 (Hz)                                        │   │
│   │   L: 신호 레벨 수                                       │   │
│   │                                                          │   │
│   │   예: 대역폭 3,100 Hz, 2개 신호 레벨                    │   │
│   │   C = 2 × 3,100 × log₂(2) = 6,200 bps                   │   │
│   │                                                          │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│   샤논 공식 (잡음 있는 채널):                                   │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                                                          │   │
│   │   C = B × log₂(1 + S/N)                                  │   │
│   │                                                          │   │
│   │   C: 채널 용량 (bps)                                    │   │
│   │   B: 대역폭 (Hz)                                        │   │
│   │   S/N: 신호 대 잡음비 (Signal-to-Noise Ratio)           │   │
│   │                                                          │   │
│   │   SNR(dB) = 10 × log₁₀(S/N)                             │   │
│   │                                                          │   │
│   │   예: 대역폭 1 MHz, SNR 63 (18 dB)                      │   │
│   │   C = 1,000,000 × log₂(1 + 63)                          │   │
│   │   C = 1,000,000 × 6 = 6 Mbps                            │   │
│   │                                                          │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘

8. 연습 문제¶

기초 문제¶

1. 물리 계층의 주요 기능 3가지를 설명하시오.

2. 다음 매체를 전송 거리가 긴 순서대로 나열하시오. - UTP, 싱글모드 광섬유, 동축 케이블, 멀티모드 광섬유

3. Cat5e와 Cat6 케이블의 차이점을 설명하시오.

4. 스트레이트 케이블과 크로스오버 케이블의 용도를 각각 설명하시오.

응용 문제¶

5. 다음 상황에서 적절한 전송 매체를 선택하고 이유를 설명하시오. - (a) 사무실 내 PC 100대 연결 - (b) 두 건물 간 1km 거리 연결 - (c) 데이터센터 내 서버 간 40Gbps 연결

6. 2.4GHz와 5GHz Wi-Fi의 장단점을 비교하시오.

7. 100m 이더넷 케이블을 통해 1Gbps 데이터를 전송할 때, 전송 지연과 전파 지연을 각각 계산하시오. (케이블에서의 빛의 속도는 2x10^8 m/s로 가정)

심화 문제¶

8. 샤논 공식을 사용하여 대역폭 10MHz, SNR 1000인 채널의 최대 전송 속도를 계산하시오.

9. 광섬유가 구리 케이블보다 우수한 점을 5가지 이상 설명하시오.

10. 데이터센터에서 Cat6a 대신 광섬유를 선택하는 이유를 설명하시오.

정답 **1.** - 비트 동기화 (송수신 클럭 동기) - 비트 전송률 제어 (bps 결정) - 전송 모드 정의 (단방향/반이중/전이중) - 물리적 인터페이스 정의 (커넥터, 전압) **2.** 싱글모드 광섬유 (~100km) > 멀티모드 광섬유 (~2km) > 동축 케이블 (~500m) > UTP (100m) **3.** - Cat5e: 최대 1Gbps, 100MHz 대역폭 - Cat6: 최대 10Gbps (55m), 250MHz 대역폭 - Cat6이 더 높은 대역폭과 간섭 내성 제공 **4.** - 스트레이트: PC-스위치, 스위치-라우터 (다른 종류 장비) - 크로스오버: PC-PC, 스위치-스위치 (같은 종류 장비) **5.** - (a) Cat5e 또는 Cat6 UTP - 저렴하고 100m 이내 적합 - (b) 싱글모드 광섬유 - 장거리, EMI 면역 - (c) 멀티모드 광섬유 (OM4) 또는 싱글모드 - 고속, 데이터센터 표준 **6.** - 2.4GHz: 범위 넓음, 벽 투과 좋음, 혼잡, 속도 낮음 - 5GHz: 범위 좁음, 벽 투과 어려움, 덜 혼잡, 속도 높음 **7.** - 전송 지연 = 데이터 크기 / 대역폭 = (예: 1500바이트) / 1Gbps = 12 마이크로초 - 전파 지연 = 거리 / 속도 = 100m / (2x10^8 m/s) = 0.5 마이크로초 **8.** C = B × log2(1 + S/N) = 10,000,000 × log2(1001) ≈ 10,000,000 × 9.97 ≈ 99.7 Mbps **9.** 1. EMI 완전 면역 2. 매우 높은 대역폭 (테라비트급) 3. 장거리 전송 가능 4. 도청 어려움 (보안성) 5. 가벼운 무게 6. 부식 없음 7. 신호 감쇠 적음 **10.** - 40G/100G 이상 속도 필요 - EMI 간섭 환경 - 미래 확장성 - 장거리 연결 필요 (캐비닛 간) - 케이블 밀도 감소
